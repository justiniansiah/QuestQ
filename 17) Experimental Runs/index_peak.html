Average Waiting Time


Simulated vs Observed Values

Waiting time is the time the customer have to wait to received their food on joining the queue. Here, we compare the average waiting time from the simulation to our observed values.

Different Initialization Queues

Here, we varied the length of the initial queue (n), and ran our simulation with common random numbers to see how our average waiting time will be affected.

For this test, we used integers of 0 to 10 to initialise the queue for the simulation.

  • We used the RStream Package to get a stream of random numnbers. These were used to generate our interarrival times for the customers.
  • We ran 1 instance of the simulation for each of the 11 queue lengths, resetting the substream each time (so that all instances will use the same substream)
  • Then we skipped to the next substream and reran another instance for the 11 queue lengths again. We repeated this process 100 times, obtaining queue time values for each length.
  • Finally we took the average values for each queue length across substreams and plotted those values against the queue length

The graph is mostly linear with the exception of 0. This is also to be expected as the rate of service does not change and a larger initalization queue length would not doubt result in longer queues and by extension, longer waiting times.

It is interesting to note that there is no observable difference in wait times between initial queue size of 0 and 1

Alternative Queue

Here, we compare the traditional queue (Model 1) to the alternative queue system whereby the store serves order/payment and food processing seperately (Model 2). For each model, we run 100 independent simulations and take the average waiting time. For each corresponding simulation (ie M1_Sim1 vs M2_Sim1, M1_Sim10 vs M2_Sim10, etc.), the interarrival times for both models will be drawn from the same substream of random numbers for each corresponding simulation. Allowing us to compare the models with less varience.
